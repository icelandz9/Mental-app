รวมโค้ดของแต่ละระบบแบบ **เต็ม ไม่ตัด** พร้อมตำแหน่งไฟล์/บรรทัด และคำอธิบาย
วิธีใช้: ก็อปแต่ละบล็อกไปวาง **carbon.now.sh** (ได้รูปโค้ดสวยมีสี) หรือวางใน Word ฟอนต์ Consolas

## 2. ระบบสมัครสมาชิก
📄 `lib/login/register/register_page.dart`

```dart
Future<void> register() async {
  if (_passwordController.text.trim() !=
      _confirmPasswordController.text.trim()) {
    ScaffoldMessenger.of(
      context,
    ).showSnackBar(const SnackBar(content: Text("รหัสผ่านไม่ตรงกัน")));
    return;
  }
  try {
    final cred = await FirebaseAuth.instance.createUserWithEmailAndPassword(
      email: _emailController.text.trim(),
      password: _passwordController.text.trim(),
    );

    // บันทึกชื่อที่กรอกตอนสมัคร ลงโปรไฟล์ users/{uid}
    await FirebaseFirestore.instance
        .collection('users')
        .doc(cred.user!.uid)
        .set({'name': _nameController.text.trim()}, SetOptions(merge: true));

    if (!mounted) return;
    ScaffoldMessenger.of(
      context,
    ).showSnackBar(const SnackBar(content: Text("สมัครสมาชิกสำเร็จ")));
    Navigator.pop(context);
  } on FirebaseAuthException catch (e) {
    if (!mounted) return;
    ScaffoldMessenger.of(
      context,
    ).showSnackBar(SnackBar(content: Text(e.message ?? "เกิดข้อผิดพลาด")));
  }
}
```

**การทำงาน:** เมื่อกดปุ่มสมัครสมาชิก ระบบจะ (1) ตรวจว่ารหัสผ่านกับช่องยืนยันตรงกันหรือไม่ (2) สร้างบัญชีใหม่ด้วย `createUserWithEmailAndPassword` ซึ่งได้ `uid` กลับมา (3) บันทึกชื่อลง Firestore ที่ `users/{uid}` โดยใช้ `merge` เพื่อไม่ทับข้อมูลอื่น ถ้าเกิดข้อผิดพลาด เช่น อีเมลซ้ำ จะถูกดักจับด้วย `FirebaseAuthException` แล้วแจ้งเตือน

```dart
Future<void> saveProfile() async {
  try {
    String uid = FirebaseAuth.instance.currentUser!.uid;
    await FirebaseFirestore.instance.collection('users').doc(uid).set({

    ScaffoldMessenger.of(

    ).showSnackBar(const SnackBar(content: Text('บันทึกข้อมูลสำเร็จ')));
  } catch (e) {
    debugPrint("Save Profile Error: $e");
  }
}

// ----- โหลดข้อมูลโปรไฟล์ -----
Future<void> loadProfile() async {
  try {
    String uid = FirebaseAuth.instance.currentUser!.uid;
    DocumentSnapshot doc = await FirebaseFirestore.instance
        .collection('users')
        .doc(uid)
        .get();
    if (doc.exists) {
      Map<String, dynamic> data = doc.data() as Map<String, dynamic>;
      setState(() {
        nameController.text = data['name'] ?? '';
        ageController.text = data['age'] ?? '';
        phoneController.text = data['phone'] ?? '';
        underlyingDiseaseController.text = data['underlyingDisease'] ?? '';
        dateController.text = data['birthday'] ?? '';
        selectedGender = data['gender'];
      });
    }
  } catch (e) {
    debugPrint("Load Profile Error: $e");
  }
}
```

**การทำงาน:** มี 2 ฟังก์ชันคู่กัน — `saveProfile` บันทึกข้อมูลที่กรอก (ชื่อ เพศ อายุ ฯลฯ) ลง `users/{uid}` โดยใช้ `merge` เพื่อไม่ทับข้อมูลเดิม ส่วน `loadProfile` อ่านข้อมูลกลับมาเติมในฟอร์มอัตโนมัติเมื่อเปิดหน้า ทั้งคู่อ้างอิงผู้ใช้ผ่าน `currentUser.uid` ให้แต่ละคนเห็นเฉพาะข้อมูลตนเอง

## 5. ระบบทำแบบทดสอบ (เก็บความคืบหน้า + เปลี่ยนข้อ)
📄 `lib/screens/question_screen.dart`

```dart
// โหลดคำตอบ/ข้อที่ค้างไว้ (ถ้ามีและจำนวนข้อตรงกัน)
Future<void> _loadProgress() async {
  final prefs = await SharedPreferences.getInstance();
  final saved = prefs.getStringList(_kAnswers);
  if (saved == null || saved.length != questions.length) return;

  final idx = prefs.getInt(_kIndex) ?? 0;
  if (!mounted) return;
  setState(() {
    for (int i = 0; i < saved.length; i++) {
      _answers[i] = int.tryParse(saved[i]) ?? -1;
    }
    _currentIndex = (idx >= 0 && idx < questions.length) ? idx : 0;
  });
}

// บันทึกความคืบหน้าปัจจุบันลงเครื่อง
Future<void> _saveProgress() async {
  final prefs = await SharedPreferences.getInstance();
  await prefs.setStringList(
    _kAnswers,
    _answers.map((e) => e.toString()).toList(),
  );
  await prefs.setInt(_kIndex, _currentIndex);
}

// กดปุ่ม "ถัดไป"
void _onNext() {
  if (_answers[_currentIndex] == -1) {
    ScaffoldMessenger.of(context).showSnackBar(
      const SnackBar(
        content: Text('กรุณาเลือกคำตอบก่อนไปข้อถัดไป'),
        duration: Duration(seconds: 2),
        behavior: SnackBarBehavior.floating,
      ),
    );
    return;
  }
  if (_currentIndex == questions.length - 1) {
    _clearProgress(); // ทำเสร็จแล้ว ล้างความคืบหน้า
    Navigator.pushReplacement(
      context,
      MaterialPageRoute(
        builder: (_) => ResultScreen(answers: List.from(_answers)),
      ),
    );
    return;
  }
  setState(() {
    _dir = 1;
    _currentIndex++;
  });
  _saveProgress();
}

// เลือกคำตอบ → เลื่อนไปข้อถัดไปเองอัตโนมัติ
void _selectAnswer(int index) {
  final int q = _currentIndex;
  setState(() => _answers[q] = index);
  _saveProgress(); // เก็บคำตอบทันที กันเผลอกดออก

  if (q == questions.length - 1) return; // ข้อสุดท้ายไม่เลื่อนเอง

  Future.delayed(const Duration(milliseconds: 250), () {
    if (!mounted || _currentIndex != q) return;
    setState(() {
      _dir = 1;
      _currentIndex++;
    });
    _saveProgress();
  });
}
```

**การทำงาน:** ควบคุมการทำแบบทดสอบทีละข้อ — `_saveProgress`/`_loadProgress` บันทึก/โหลดคำตอบที่ค้างไว้ลงเครื่อง (SharedPreferences) กันเผลอปิดแอปกลางคัน `_onNext` ตรวจว่าตอบข้อปัจจุบันแล้วหรือยังก่อนไปข้อถัดไป ข้อสุดท้ายจะพาไปหน้าผลลัพธ์ ส่วน `_selectAnswer` บันทึกคำตอบทันทีแล้วเลื่อนไปข้อถัดไปอัตโนมัติ (หน่วง 250 ms ให้เห็นสถานะที่เลือกก่อน)

## 6. คิดคะแนนรวมแยกตามแบบทดสอบ
📄 `lib/screens/result_screen.dart`

```dart
List<_ScoreResult> _computeAll() {
  const order = ['PHQ-9', 'GAD-7', 'PSS-10', 'ASRS', 'OCI-R', 'PCL-5', 'MDQ'];
  final out = <_ScoreResult>[];

  for (final inst in order) {
    int sum = 0; // คะแนนรวม (PHQ-9/GAD-7/PSS/OCI-R/PCL-5)
    int partA = 0; // จำนวนข้อ Part A ที่ถึงเกณฑ์ (ASRS)
    int yes = 0; // จำนวน "ใช่" (MDQ)

    for (int i = 0; i < questions.length; i++) {
      final q = questions[i];
      if (q.instrument != inst) continue;
      final a = widget.answers[i];
      final s = (a == -1) ? 0 : q.scores[a];
      sum += s;
      if (q.threshold != null && a != -1 && q.scores[a] >= q.threshold!) {
        partA++;
      }
      if (inst == 'MDQ' && s == 1) yes++;
    }

    out.add(_interpret(inst, sum: sum, partA: partA, yes: yes));
  }
  return out;
}
```

**การทำงาน:** คิดคะแนนของแบบทดสอบทั้ง 7 ชุดในครั้งเดียว โดยวนอ่านทุกคำถามในแต่ละชุดแล้วรวมคะแนนคำตอบที่เลือก (`sum`) พร้อมเก็บค่าพิเศษ เช่น `partA` (นับข้อที่ถึงเกณฑ์ของ ASRS) และ `yes` (นับคำตอบ "ใช่" ของ MDQ) จากนั้นส่งไปแปลผลด้วย `_interpret`

```dart
  required int yes,

        flagged: flagged,

    case 'PCL-5':
      final flagged = sum >= 31;
      return _ScoreResult(
        instrument: 'PCL-5',

        score: sum,

// ตัวช่วยแปลผลแบบช่วงคะแนน (band)
_ScoreResult _band(
  String inst,
  String cond,
  int score,
  int max,
  List<List<dynamic>> bands,
) {
  for (final b in bands) {
    if (score <= (b[0] as int)) {
      return _make(inst, cond, score, max, b);
    }
  }
  return _make(inst, cond, score, max, bands.last);
}

_ScoreResult _make(
  String inst,
  String cond,
  int score,
  int max,
  List<dynamic> b,
) {
  final level = b[1] as String;
  final flagged = b[3] as bool;
  return _ScoreResult(
    instrument: inst,
    condition: cond,
    score: score,
    max: max,
    level: level,
    color: b[2] as Color,
    flagged: flagged,
    advice: flagged
        ? 'อยู่ในระดับ"$level" แนะนำปรึกษาจิตแพทย์/นักจิตวิทยา'
        : 'อยู่ในระดับ"$level" ดูแลสุขภาพใจต่อไปได้',
  );
}
```

**การทำงาน:** แปลคะแนนดิบเป็นระดับความรุนแรงตามเกณฑ์มาตรฐาน โดยใช้ `switch` แยกตามแบบทดสอบ — กลุ่มช่วงคะแนน (PHQ-9, GAD-7, PSS-10) ส่งตารางช่วงให้ `_band` เลือกระดับ ส่วนกลุ่มจุดตัด (ASRS, OCI-R, PCL-5, MDQ) เทียบคะแนนกับ cutoff โดยตรงว่า "มีแนวโน้ม" หรือไม่ และ `_make` สร้างผลลัพธ์พร้อมกำหนดสีและคำแนะนำอัตโนมัติ

```dart
Future<void> _saveToFirebase() async {
```

## 10. กฎความปลอดภัยของฐานข้อมูล
📄 `firestore.rules`

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // อีเมลแอดมิน
    function isAdmin() {
      return request.auth != null
        && request.auth.token.email in ['admin@gmail.com'];
    }

    match /users/{uid} {
      // อ่านได้เฉพาะเจ้าของ หรือแอดมิน
      allow read: if request.auth != null
        && (request.auth.uid == uid || isAdmin());
      allow write: if request.auth != null && request.auth.uid == uid;

      match /results/{resultId} {
        allow read: if request.auth != null
          && (request.auth.uid == uid || isAdmin());
        allow create: if request.auth != null && request.auth.uid == uid;
        allow update, delete: if false; // ห้ามแก้/ลบผลย้อนหลัง
      }
    }

    // อนุญาต collectionGroup query เฉพาะแอดมิน
    match /{path=**}/results/{resultId} {
      allow read: if isAdmin();
    }
  }
}
```

**การทำงาน:** กำหนดสิทธิ์เข้าถึงข้อมูลเพื่อปกป้องข้อมูลสุขภาพจิตที่อ่อนไหว — ผู้ใช้ทั่วไปอ่าน/แก้ได้เฉพาะข้อมูลตนเอง (`request.auth.uid == uid`) ส่วนแอดมิน (ตรวจจากอีเมลใน `isAdmin`) อ่านข้อมูลทุกคนได้เพื่อใช้กับเว็บแดชบอร์ด และผลที่บันทึกแล้วห้ามแก้/ลบ (`update, delete: if false`) เพื่อรักษาความถูกต้องย้อนหลัง

## 11. เว็บแดชบอร์ดแอดมิน — ดึงข้อมูลผู้ใช้ทุกคน
📄 `admin/app.js`

```javascript
async function loadData() {
  // 1) อ่านโปรไฟล์ผู้ใช้ทุกคน
  const usersSnap = await getDocs(collection(db, "users"));
  const profiles = {};
  usersSnap.forEach((d) => (profiles[d.id] = d.data()));

  // 2) อ่านผลประเมินของทุกคนพร้อมกันด้วย collectionGroup
  const resSnap = await getDocs(collectionGroup(db, "results"));
  const resultsByUid = {};
  resSnap.forEach((d) => {
    const uid = d.ref.parent.parent.id; // หา uid เจ้าของผล
    (resultsByUid[uid] ||= []).push({ id: d.id, ...d.data() });
  });

  // 3) รวมข้อมูลรายคน แล้วนำไปแสดง
  const uids = new Set([
    ...Object.keys(profiles),
    ...Object.keys(resultsByUid),
  ]);
  allUsers = [...uids].map((uid) => ({
    uid,
    profile: profiles[uid] || {},
    results: resultsByUid[uid] || [],
  }));

  renderStats();
  renderList(allUsers);
}
```

**การทำงาน:** เว็บแดชบอร์ดแอดมิน (แยกจากแอปหลัก) ใช้ Firebase JS SDK ทำ 3 ขั้นตอน — (1) อ่านโปรไฟล์ทุกคนจาก collection `users` (2) อ่านผลประเมินทุกคนพร้อมกันด้วย `collectionGroup` แล้วหาเจ้าของจาก `parent.parent.id` (3) รวมโปรไฟล์กับผลประเมินเป็นรายคนแล้วแสดงเป็นการ์ด ทั้งนี้อ่านข้อมูลทุกคนได้เฉพาะบัญชีแอดมินตามกฎความปลอดภัย
