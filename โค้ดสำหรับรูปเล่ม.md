## 2. ระบบสมัครสมาชิก (register_page.dart)

```dart
Future<void> register() async {
  // ตรวจว่ารหัสผ่านกับยืนยันรหัสผ่านตรงกันหรือไม่
  if (_passwordController.text.trim() !=
      _confirmPasswordController.text.trim()) {
    showSnackBar("รหัสผ่านไม่ตรงกัน");
    return;
  }
  try {
    // สร้างบัญชีใหม่ด้วยอีเมล + รหัสผ่าน ผ่าน Firebase Authentication
    await FirebaseAuth.instance.createUserWithEmailAndPassword(
      email: _emailController.text.trim(),
      password: _passwordController.text.trim(),
    );
    Navigator.pop(context); // สมัครสำเร็จ → กลับไปหน้าเข้าสู่ระบบ
  } on FirebaseAuthException catch (e) {
    showSnackBar(e.message ?? "เกิดข้อผิดพลาด");
  }
}
```

**อธิบาย:** รับข้อมูลจากผู้ใช้ ตรวจความถูกต้องของรหัสผ่าน แล้วสร้างบัญชีใหม่ผ่าน Firebase Authentication หากอีเมลซ้ำหรือรหัสผ่านไม่ผ่านเกณฑ์จะแจ้งเตือน

## 3. การทำแบบทดสอบ (question_screen.dart)

```dart
void _onNext() {
  // ยังไม่เลือกคำตอบ → เตือนให้เลือกก่อน
  if (_answers[_currentIndex] == -1) {
    showSnackBar('กรุณาเลือกคำตอบก่อนไปข้อถัดไป');
    return;
  }
  // ถ้าเป็นข้อสุดท้าย → ไปหน้าแสดงผลลัพธ์
  if (_currentIndex == questions.length - 1) {

      builder: (_) => ResultScreen(answers: List.from(_answers))));
    return;

  setState(() => _currentIndex++); // ไปข้อถัดไป
  _saveProgress();                 // บันทึกความคืบหน้าไว้ในเครื่อง
}
```

## 4. การคิดคะแนนรวมแยกตามแบบทดสอบ (result_screen.dart)

```dart
List<_ScoreResult> _computeAll() {
  const order = ['PHQ-9', 'GAD-7', 'PSS-10', 'ASRS', 'OCI-R', 'PCL-5', 'MDQ'];
  final out = <_ScoreResult>[];

  for (final inst in order) {
    int sum = 0;    // คะแนนรวม
    int partA = 0;  // จำนวนข้อที่ถึงเกณฑ์ (ใช้กับ ASRS)
    int yes = 0;    // จำนวนคำตอบ "ใช่" (ใช้กับ MDQ)

    // วนอ่านทุกคำถามของแบบทดสอบนี้ แล้วรวมคะแนนจากคำตอบที่ผู้ใช้เลือก
    for (int i = 0; i < questions.length; i++) {
      final q = questions[i];
      if (q.instrument != inst) continue;
      final a = widget.answers[i];
      final s = (a == -1) ? 0 : q.scores[a];
      sum += s;
      if (q.threshold != null && a != -1 && q.scores[a] >= q.threshold!) partA++;
      if (inst == 'MDQ' && s == 1) yes++;
    }
    out.add(_interpret(inst, sum: sum, partA: partA, yes: yes)); // แปลผล
  }
  return out;
}
```

**อธิบาย:** หัวใจของระบบ — นำคำตอบทั้งหมดมาแยกคิดคะแนนตามแบบทดสอบมาตรฐานทั้ง 7 ชุด (PHQ-9, GAD-7 ฯลฯ) แล้วส่งต่อไปแปลผลในขั้นถัดไป

## 5. การแปลผลตามเกณฑ์มาตรฐาน (result_screen.dart)

```dart
_ScoreResult _interpret(String inst, {required int sum, ...}) {
  switch (inst) {
    case 'PHQ-9': // ภาวะซึมเศร้า — แบ่งระดับความรุนแรงตามช่วงคะแนน
      return _band('PHQ-9', 'ภาวะซึมเศร้า', sum, 27, const [
        [4,  'ปกติ / น้อยมาก',  _cGreen,      false],
        [9,  'เล็กน้อย',         _cLime,       false],
        [14, 'ปานกลาง',          _cOrange,     true],
        [19, 'ค่อนข้างรุนแรง',    _cDeepOrange, true],
        [27, 'รุนแรง',            _cRed,        true],
      ]);

    case 'OCI-R': // โรคย้ำคิดย้ำทำ — ใช้จุดตัด (cutoff) ที่ 21 คะแนน
      final flagged = sum >= 21;
      return _ScoreResult(
        instrument: 'OCI-R', condition: 'โรคย้ำคิดย้ำทำ (OCD)',
        score: sum, max: 72,
        level: flagged ? 'มีแนวโน้ม' : 'ต่ำกว่าเกณฑ์',
        flagged: flagged,
      );

    // แบบทดสอบอื่น (GAD-7, PSS-10, ASRS, PCL-5, MDQ) ใช้หลักการเดียวกัน
  }
}
```

**อธิบาย:** แปลคะแนนดิบให้เป็นระดับความรุนแรงตามเกณฑ์ทางคลินิกของแต่ละแบบทดสอบ เช่น PHQ-9 แบ่งเป็น 5 ระดับ ส่วน OCD/PTSD ใช้ "จุดตัด" (ตัวอย่างในเล่มขอแสดง 2 แบบพอ ที่เหลือใช้หลักการเดียวกัน)

## 6. การบันทึกผลลงฐานข้อมูล (result_screen.dart)

```dart
Future<void> _saveToFirebase() async {
  final user = FirebaseAuth.instance.currentUser;
  if (user == null) return;

  // บันทึกผลไว้ใต้ users/{uid}/results เพื่อแยกข้อมูลตามผู้ใช้แต่ละคน
  await FirebaseFirestore.instance
      .collection('users').doc(user.uid).collection('results')
      .add({
        'createdAt': FieldValue.serverTimestamp(), // เวลาที่ทำ
        'flaggedCount': _flaggedCount,             // จำนวนด้านที่ควรใส่ใจ
        'selfHarmRisk': _selfHarmRisk,             // สัญญาณเสี่ยงทำร้ายตัวเอง
        'results': _results.map((r) => {
          'instrument': r.instrument, 'condition': r.condition,
          'score': r.score, 'max': r.max,
          'level': r.level, 'flagged': r.flagged,
        }).toList(),
      });
}
```

**อธิบาย:** บันทึกผลการประเมินลงฐานข้อมูล Firestore โดยเก็บแยกตามผู้ใช้ (uid) เพื่อให้ย้อนดูประวัติได้และรักษาความเป็นส่วนตัว

## 8. กฎความปลอดภัยของฐานข้อมูล (firestore.rules)

```javascript
match /users/{uid} {
  // อ่านได้เฉพาะเจ้าของข้อมูล หรือผู้ดูแล (แอดมิน)
  allow read: if request.auth != null
    && (request.auth.uid == uid || isAdmin());
  allow write: if request.auth != null && request.auth.uid == uid;

  match /results/{resultId} {
    allow read: if request.auth != null
      && (request.auth.uid == uid || isAdmin());
    allow create: if request.auth != null && request.auth.uid == uid;
    allow update, delete: if false; // ห้ามแก้/ลบผลย้อนหลัง
  }
}
```

**อธิบาย:** กำหนดสิทธิ์การเข้าถึงข้อมูล — ผู้ใช้ทั่วไปเห็นเฉพาะข้อมูลของตนเอง แอดมินเห็นได้ทุกคน และผลการประเมินที่บันทึกแล้วห้ามแก้ไข/ลบ เพื่อความปลอดภัยของข้อมูลสุขภาพจิตที่อ่อนไหว

## 9. เว็บแดชบอร์ดแอดมิน — ดึงข้อมูลผู้ใช้ทุกคน (admin/app.js)

```javascript
async function loadData() {
  // 1) อ่านโปรไฟล์ผู้ใช้ทุกคน
  const usersSnap = await getDocs(collection(db, "users"));
  const profiles = {};
  usersSnap.forEach(d => profiles[d.id] = d.data());

  // 2) อ่านผลประเมินของทุกคนพร้อมกันด้วย collectionGroup
  const resSnap = await getDocs(collectionGroup(db, "results"));
  const resultsByUid = {};
  resSnap.forEach(d => {
    const uid = d.ref.parent.parent.id;      // หา uid เจ้าของผล
    (resultsByUid[uid] ||= []).push(d.data());
  });

  renderList(profiles, resultsByUid); // นำไปแสดงเป็นการ์ดรายคน
}
```

**อธิบาย:** เว็บแยกสำหรับผู้ดูแล ใช้ Firebase JS SDK อ่านโปรไฟล์และผลประเมินของผู้ใช้ทุกคน (ผ่านคำสั่ง collectionGroup) มาแสดงรวมในหน้าเดียว
